## 项目主要需求

### 项目概述
**项目类型：纯后端服务**
搭建一个二手循环商店后端 API 服务，支持用户发布闲置物品、浏览购买、交易管理和环保积分体系。

### 功能需求

1. **商品管理**：用户发布二手商品（名称、描述、原价、售价、成色、分类（数码/服饰/图书/家居/母婴/运动）、图片），支持上下架管理和库存标记。
2. **商品浏览与搜索**：按分类、价格区间、成色、发布时间筛选商品，支持关键词全文搜索，结果按相关度/价格/新旧排序。
3. **购物车与订单**：加入购物车、批量下单，订单状态流转（待付款→待发货→已发货→已收货→已完成/退款中→已退款）。
4. **环保积分体系**：每次成功交易双方获得环保积分，积分计算规则基于商品重量/类别，积分可兑换优惠券。
5. **评价与信用**：交易完成后互评，好评率影响用户信用等级，信用等级影响商品曝光权重。
6. **消息通知**：系统通知（订单状态变更、积分到账、新订单提醒），通知状态管理（已读/未读）。
7. **数据统计**：平台交易统计接口（日/周/月交易量、交易额、热门分类排行、用户活跃度）。

### 技术要求

- 后端框架：Django + Django REST Framework + Python
- 数据库：PostgreSQL
- 认证：JWT（djangorestframework-simplejwt）
- 搜索：PostgreSQL 全文搜索（SearchVector + SearchRank）
- 图片存储：Django media 目录
- 代码规范：严格遵循单一职责原则，每个文件只负责一个功能；常量（错误码、订单状态枚举、积分规则）统一维护在 `backend/app/constants/` 目录下；日志统一使用 Python logging 模块，日志配置在 `backend/app/utils/logger.py`；异常信息统一定义在 `backend/app/constants/errors.py`，后端通过 DRF 自定义异常处理统一返回标准格式响应。

## 部署要求

项目必须支持 Docker Compose 一键部署。

需要提供以下文件：
- 根目录 `docker-compose.yml`（编排后端 + 数据库）
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=cycleapi`
- `backend/Dockerfile`
- `.env.example`

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: cycleapi`
- 容器名带 `cycleapi-` 前缀
- 数据库数据使用命名卷持久化
- 数据库配置 `healthcheck`
- 端口映射：
  - 后端：19413:8000
- 访问 API 文档（DRF Browsable API）：http://localhost:19413/api/
